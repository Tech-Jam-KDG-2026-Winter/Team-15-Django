## 1. はじめに：このガイドについて
このガイドは、限られた時間の中で効率的にチーム開発を進めるためのものです。僕（リーダー）が提供する「アプリ全体の要件定義」と「クラス設計概要」も合わせて確認し、開発の全体像を理解してください。

## 2. プロジェクト概要と私たちの目標
僕たちが目指すのは、ユーザーの体調に応じたパーソナルな運動提案を通じて、健康的なライフスタイルの定着をサポートするアプリです。

<a id="3"></a>
## 3. 開発環境のセットアップ
Gitからプロジェクトをクローンしたら、まず以下の手順で開発環境を整えてください。

1. **Python仮想環境の構築と有効化(これに関してはおそらくみなさんは終えているはずなのでやらなくても大丈夫です)**:
    ```bash
    python3 -m venv venv
    source venv/bin/activate 
    # Windowsの場合: .\venv\Scripts\activate
    ```
    ※新しいターミナルセッションを開くたびに `source venv/bin/activate` を実行してください。

2. **必要なPythonライブラリのインストール**:
    ```bash
    pip install -r requirements.txt
    ```

3. **データベースの初期化とマイグレーションの適用**:
    ```bash
    ./venv/bin/python manage.py makemigrations
    ./venv/bin/python manage.py migrate
    ```

4. **管理者（スーパーユーザー）アカウントの作成**:
    ```bash
    ./venv/bin/python manage.py createsuperuser
    ```

5. **開発サーバーの起動**:
    ```bash
    ./venv/bin/python manage.py runserver
    ```

<a id="4"></a>
## 4. Gitワークフローとコード共有
*   **ブランチ戦略**: `main` ブランチから自分の担当機能に応じたブランチを作成してください。
    *   例: `feature/hirota-recommend-api`
    *   例: `feature/suzuki-routine-history`
    *   例: `feature/sugiyama-top-ui`
*   **コミットメッセージ**: 変更内容が分かりやすいメッセージを心がけましょう。
*   **プルリクエスト (PR)**: 作業が一段落したら、`main` ブランチに向けてPRを作成し、僕がレビュー・マージを行います。

<a id="5"></a>
## 5. リーダー（自分）の役割
僕が以下の部分を担当し、皆さんの作業がスムーズに進むようサポートします。

*   `static/js/main.js` 内でのJavaScriptによるAPI連携（`fetch`の呼び出しや、データの渡し）
*   `main.js` 内の `display...` 関数や `addRoutine`/`deleteRoutine` といった、各機能のJavaScriptの繋ぎ込み
*   全体の進捗管理、技術的な難しい問題の解決、最終的なコードの統合

皆さんは、APIのロジック開発や画面のUI実装（DOM操作）に集中してください。

## 補足：実装中の「動作確認」について

今回の開発では、API連携（非同期通信）は最終的に僕がまとめて実装します。
そのため、皆さんが担当部分を実装している段階では、

・画面がまだ更新されない  
・ボタンを押しても何も起きない  
・ダミーデータ前提の状態  

になることがありますが、**それは正常です**。

皆さんは  
「HTMLの構造が正しいか」  
「指定されたid / classが付いているか」  
「PythonのAPIが仕様どおりのレスポンスを返すか」  
を確認できていればOKです。
